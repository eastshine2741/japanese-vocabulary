# 홈 화면 개편 기획서

## 목적

유저 리텐션 향상, 노래 재생 또는 플래시카드 리뷰로의 인입 향상.

## 현재 상태와 문제점

현재 홈 화면은 검색 바 + "최근 들은 노래" 3열 격자(최대 15곡)로 구성되어 있다.

| 문제 | 영향 |
|---|---|
| 행동 유도가 없다 | 앱을 열어도 "지금 뭘 해야 하는지" 모른다. 검색 바와 앨범 썸네일만 보인다 |
| 복귀 동기가 없다 | 매일 돌아올 이유(streak, 진행도)가 화면에 없다 |
| 빈 화면이 막다른 길 | 신규 유저에게 "아직 들은 노래가 없어요" 이후 할 수 있는 행동이 검색뿐이다 |

## 단어 탭과의 역할 분리

단어 탭은 이미 복습 예정 수, 단어 상태 breakdown, 노래별 단어장을 상세하게 보여주고 있다.
홈에 같은 정보를 중복하지 않고, **깊이를 달리하여** 역할을 분리한다.

| | 홈 탭 | 단어 탭 |
|---|---|---|
| **역할** | Trigger — "오늘 뭐 해야 해?" | Workspace — "내 단어 현황과 관리" |
| **복습 예정** | 숫자 + CTA 버튼 (compact) | 큰 숫자 + 상태 bar + legend + CTA |
| **노래별 현황** | 없음 (단어 탭에 위임) | 노래별 단어장 리스트 (due badge 포함) |
| **진입 깊이** | 1 tap → 바로 학습 시작 | 노래 선택 → 단어 목록 → 학습 |

## S1: 오늘의 학습 액션 카드

홈 화면의 핵심 CTA. 유저 상태에 따라 내용이 달라진다.

### 상태별 표시

| 우선순위 | 조건 | 카드 내용 | CTA |
|:---:|---|---|---|
| 1 | 복습 예정 카드 > 0 | "복습할 단어 N개" | 지금 복습하기 → 플래시카드 리뷰 화면 |
| 2 | 복습 예정 = 0, 저장 단어 > 0 | "오늘 복습 완료!" | (축하 상태, CTA 없음) |
| 3 | 저장 단어 = 0, 최근 노래 > 0 | "노래를 들으며 단어를 저장해보세요" | 학습하기 → 마지막 노래 재생 |
| 4 | 완전 신규 유저 | "좋아하는 일본 노래로 시작하기" | 노래 검색하기 → 검색 화면 |

### Acceptance Criteria

- [ ] 복습 예정 카드 수(`/api/flashcards/stats`의 `due`)에 따라 상태가 결정된다
- [ ] 상태 1: "복습할 단어 N개" 텍스트와 "지금 복습하기" 버튼이 표시된다
- [ ] 상태 1: 버튼 탭 시 플래시카드 리뷰 화면(전체 덱)으로 이동한다
- [ ] 상태 2: "오늘 복습 완료!" 텍스트와 체크 아이콘이 표시된다
- [ ] 상태 3: 마지막 재생 노래의 썸네일과 함께 학습 유도 메시지가 표시된다
- [ ] 상태 4: 검색 유도 메시지와 버튼이 표시된다
- [ ] 홈 탭 포커스 시 매번 최신 상태를 불러온다

### 필요 데이터

기존 API로 충분: `/api/flashcards/stats` (due, total), `/api/songs/recent`

## S2: 최근 들은 노래

기존 세로 스크롤 격자를 **페이지형 가로 스크롤**로 변경한다.

### 변경사항

| 항목 | 기존 | 변경 |
|---|---|---|
| 표시 곡 수 | 최대 15곡 | 최대 18곡 (9곡 x 2페이지) |
| 레이아웃 | 세로 스크롤 3열 격자 | 3열 x 3행 격자를 가로 스크롤 (snap paging) |
| 페이지 인디케이터 | pagination dots (페이지 이동 없음) | 현재 페이지를 나타내는 dots (2개) |

```
페이지 1                    페이지 2
┌───┐ ┌───┐ ┌───┐        ┌───┐ ┌───┐ ┌───┐
│   │ │   │ │   │   ←→   │   │ │   │ │   │
└───┘ └───┘ └───┘        └───┘ └───┘ └───┘
┌───┐ ┌───┐ ┌───┐        ┌───┐ ┌───┐ ┌───┐
│   │ │   │ │   │   ←→   │   │ │   │ │   │
└───┘ └───┘ └───┘        └───┘ └───┘ └───┘
┌───┐ ┌───┐ ┌───┐        ┌───┐ ┌───┐ ┌───┐
│   │ │   │ │   │   ←→   │   │ │   │ │   │
└───┘ └───┘ └───┘        └───┘ └───┘ └───┘
        ● ○                     ○ ●
```

### Acceptance Criteria

- [ ] 3열 x 3행 격자가 한 페이지를 구성한다 (페이지당 9곡)
- [ ] 가로 스크롤로 최대 2페이지를 넘길 수 있다 (최대 18곡)
- [ ] 페이지 단위 snap scrolling이 적용된다
- [ ] 현재 페이지를 나타내는 dots가 격자 아래에 표시된다
- [ ] 기존 SongCard 컴포넌트를 그대로 사용한다
- [ ] 9곡 이하이면 1페이지만 표시되고 dots는 숨겨진다
- [ ] 최근 노래가 없으면 기존 empty state를 표시한다

## 빈 화면 시나리오

| 유저 상태 | S1 (액션 카드) | S2 (최근 노래) |
|---|---|---|
| 완전 신규 | 상태 4: 검색 유도 | empty state |
| 노래 들었지만 단어 저장 전 | 상태 3: 학습 유도 | 최근 노래 표시 |
| 단어 저장, 오늘 복습 완료 | 상태 2: 완료 축하 | 최근 노래 표시 |
| 복습 예정 있음 | 상태 1: 복습 CTA | 최근 노래 표시 |

## Streak 카드

[학습 통계 기획서](./study-stats-user-stories.md)의 US-1에서 별도로 정의.
검색 바와 S1 사이에 배치되며, 이 기획의 구현과 독립적으로 진행 가능.

## 구현 우선순위

S1과 S2는 하나의 작업 단위로 함께 진행한다.

| 순위 | 섹션 | 리텐션 기여 | 이유 |
|:---:|---|:---:|---|
| P0 | S1: 오늘의 학습 액션 카드 | 매우 높음 | 앱을 열 때마다 명확한 행동을 제시. 플래시카드 인입의 핵심 경로 |
| P0 | S2: 최근 노래 축소 | - | S1 추가에 따른 레이아웃 조정. 독립 가치는 없지만 S1과 함께 진행 |
